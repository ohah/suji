# autoUpdater

Electron `autoUpdater` / Tauri `updater`에 대응하는 1차 API. 현재 범위는
업데이트 manifest를 확인해 새 버전 여부를 판단하고, 별도 다운로드된 artifact의
SHA-256을 검증하는 안전 프리미티브다. 패키지 교체, quit-and-install, 재시작 적용은
아직 후속이다.

## Manifest

```json
{
  "version": "1.2.3",
  "url": "https://releases.example.com/my-app-1.2.3.dmg",
  "sha256": "0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef",
  "notes": "Bug fixes",
  "pubDate": "2026-05-25T00:00:00Z"
}
```

`url`은 `https://`, `http://`, `file://`만 허용한다. 로컬 테스트에는 `file://`를
쓸 수 있지만 배포 manifest는 HTTPS와 SHA-256을 함께 쓰는 것을 권장한다.

## Frontend

```ts
import { autoUpdater } from "@suji/api";

const check = await autoUpdater.checkForUpdates("https://example.com/update.json");
if (check.updateAvailable) {
  // 앱 정책에 맞춰 artifact를 다운로드한 뒤 검증한다.
  const verified = await autoUpdater.verifyFile("/tmp/my-app.dmg", check.sha256);
}
```

manifest 객체를 직접 넘길 수도 있다.

```ts
await autoUpdater.checkForUpdates({
  version: "1.2.3",
  url: "https://example.com/my-app.zip",
  sha256: "0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef"
}, { currentVersion: "1.0.0" });
```

## Node Backend

```ts
import { autoUpdater } from "@suji/node";

const r = await autoUpdater.checkForUpdates({
  version: "1.2.3",
  url: "https://example.com/my-app.zip",
});
```

## `__core__` cmd

| cmd | 입력 | 응답 |
|---|---|---|
| `auto_updater_check_update` | `currentVersion`, `latestVersion` 또는 `version`, `url`, optional `sha256`, `notes`, `pubDate` | `{success:true, updateAvailable, currentVersion, version, url, sha256, notes, pubDate}` |
| `auto_updater_verify_file` | `path`, `sha256` | `{success:boolean, actualSha256}`. 읽기 실패/invalid hash는 `{success:false,error}` |

버전 비교는 semver 계열 숫자 segment를 비교하고, `v` prefix와 build metadata
(`+build`)는 허용한다. prerelease는 semver 규칙처럼 release보다 낮게 본다.

## 정직 경계

- 다운로드 자체와 설치 적용은 아직 제공하지 않는다. v1은 업데이트 가능 여부와
  artifact 검증만 담당한다.
- `checkForUpdates(url)`의 manifest fetch는 JS/Node 런타임의 `fetch`를 사용한다.
  네이티브 HTTP 클라이언트로 백그라운드 다운로드하는 경로는 후속이다.
- checksum 검증은 로컬 파일 전체를 읽어 SHA-256을 계산한다. 대형 파일도 스트리밍으로
  처리하지만, 파일 교체와 권한 상승 설치는 앱/패키지별 후속 정책이 필요하다.
