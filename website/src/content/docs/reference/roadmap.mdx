## 구현됨 (발췌)

- 창/렌더러 제어, clipboard(text/html/rtf/png/**tiff**/buffer), dialog,
  shell, globalShortcut, tray, menu, notification, screen, powerMonitor,
  safeStorage(macOS/Windows), session(cookies/clearStorageData), webRequest
  (blocklist + dynamic listener + **timeout fallback**), desktopCapturer
  (sources + 썸네일 파일경로).
- 멀티 백엔드(Zig·Rust·Go·Node) + 자동 라우팅, 공식 플러그인 state·sqlite
  (5언어 래퍼), `suji types` 타입 자동 생성(Zig).
- 권한 모델: fs 샌드박스 + shell/dialog/캡처/nativeImage opt-in 게이트 +
  모바일 권한 게이트(iOS/Android 실 시뮬·에뮬 e2e 검증).
- 임베드 코어 분리 + 모바일(iOS/Android, Rust·Go 백엔드, SQLite).

## 정직한 경계

- **`protocol.handle`**(임의 cross-origin 동적 핸들러): CEF IO-스레드 결함으로
  보류 — same-origin 정적 서빙은 `protocol:"suji"` 가 충족.
- **iOS 실 HTTPS/TLS**: std CA 번들 공백(시뮬·빌드 검증, 실기기 미해결).
- **desktopCapturer 썸네일 인코딩**: Screen Recording TCC 권한 필요 —
  헤드리스/CI 는 컴파일·링크·graceful-fail 만 검증(인코딩 경로 미실행).
- `process.run`/`http.fetch`: **백엔드 only**(렌더러 미노출 — 의도된 보안 설계).
- Windows/Linux 패키징·자동 업데이트·크래시 리포터: 미구현(우선순위).

테스트: `zig build test` 790개 단위 + e2e(puppeteer/bun) + 모바일 실
시뮬/에뮬 기능 e2e + 호스트 하니스. 정직한 미검증 경계는 PLAN/커밋에 명시.
